shiro学习笔记

Posted on 2019-03-13
别看了,都是辣鸡

一些相关的概念:

  • Subject:当前用户, 当前线程/session的用户
  • Realm: 是一种DAO, 是shiro(领域)的DAO
  • Principal: 当事人

  • Credential: 认证信息

  • AuthenticationInfo: subject的相关的账户的信息, 表示已验证和存储的帐户数据.

  • AuthenticationToken: 表示提交给login的任何数据, 包括成功的和不成功的.

  • AuthorizationInfo: 表示单个subject存储的授权数据(roles,permissions).

  • PrincipalCollection: 表示一个身份集合, 因为我们可以在Shiro中同时配置多个Realm,或者说一个subject可能有多个身份,
    所以身份信息可能就有多个, 因此其提供了PrincipalCollection用于聚合这些身份信息.

  • Authentication 和 Authorization:
    如何分辨两个如此相似的单词?

    • Authentication 身份认证;
      authentic：真实可信的
      【结构分析】： authentic = aut(自己) + hent(行为人) + ic(形容词后缀) →自主行为人所说的或所做的→并非屈打成招的→真实可信的
      【词源解释】： aut <-希腊语 autos (自己); hent <-希腊语 hentes (行为人)
      【衍生词】： authenticity (真实性); authenticate (鉴定); authentication (证明)
    • Authorization 授权
      authorize
      词根词缀： auth ( -auct- )增加->创造->权威 + -or 名词词尾 , 行为 + -ize 动词词尾

shiro的realm怎么写

一般情况下:

  1. 首先, extends AuthorizingRealm ;

  2. realm里面Autowired shiro相关的domian的service; 这是为了从数据源中获取用户的信息,主要是role和permission;

  3. 然后是重写两个方法: doGetAuthorizationInfo 和 doGetAuthenticationInfo ;

  4. doGetAuthenticationInfo: 是认证的方法; 就是从当前的(默认)AuthenticationToken中取出username和password
    然后从service中取出相关的用户进行验证;
    入参是AuthenticationToken,出参是AuthenticationInfo, 表示的是从未认证到已认证的转变;

  5. doGetAuthorizationInfo: 检测用户权限的时候才会调用此方法，例如checkRole,checkPermission之类的.
    入参是:PrincipalCollection,出参是:AuthorizationInfo.
    返回的是这个Subject的所有的身份 和 权限的集合;

shiro的shiroConfig怎么写

ShiroConfig主要是注入shiro的filterFactoryBean和securityManager等对象

ShiroFilterFactoryBean 处理拦截资源文件问题.
注意：单独一个ShiroFilterFactoryBean配置是会报错的, 因为在初始化ShiroFilterFactoryBean的时候需要注入SecurityManager ;

  • Filter Chain定义说明:

    1. 一个URL可以配置多个Filter,使用逗号分隔
    2. 当设置多个过滤器时，全部验证通过,才视为通过
    3. 部分过滤器可指定参数，如perms,roles
    4. 过滤链从上向下顺序匹配,一般将/**放在最为下边

  • securityManager主要是设置Realm;

其他的

shiro也可以有自己的vo. 有别于ORM的pojo, 是ORM的pojo再包装一次去掉不需要的,或者添加其他需要的属性/方法(所以说称之为vo嘛~~);
如果没有自己的vo, 就需要在设计数据库的表的时候用点心思了;
既然都可以有自己的vo了,service也是可以的.
