第一部分

IO 流分为两种: 是不是给人看的

  • 字节流: 是按照字节来读取的, 可以读取二进制文件,不是给人看的 InputStream 和 OutputStream
  • 字符流: 是基于字节流实现的, 是针对字符文件的读写所做的优化,给人看的 Writer 和 Reader

InputStream 和 OutputStream 是抽象类(abstract 修饰), 不能实例化, 只能用他们的实现类来修饰.

还可以这样分类: 按照流是否直接与特定的地方(如磁盘.内存.设备等)相连,分为节点流和处理流两类.

  • 节点流: 直接与外围设备关联的数据的源和目的地交互 如 FileInputStream
  • 处理流: 对一个已经存在的流的过滤封装等处理 如 BufferReader

节点流基本有这三个: 文件, 管道, 数组(ByteArrayInputStream).

处理流有这么几个: 缓冲操作,基本数据类型操作,对象序列化操作,转化控制,打印控制.
