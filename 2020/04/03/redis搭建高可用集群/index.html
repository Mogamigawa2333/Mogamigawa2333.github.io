redis搭建高可用集群

Posted on 2020-04-03

用料:

  • Redis 5.0 以上, 5.0以上对集群有原生支持

原理:

  • 通过redis-cli与整个集群交互, 读写键值的时候, 先通过CRC函数计算键的hash值, 再计算出所对应的槽位;

    比如: 执行 set hello world, 先计算出键hello 对应的槽位是866, 然后找到负责这个槽位的master-slave系统;

  • 每个master与slave的比例都是1:1或者1:N, 没有N:N或者N:1;

  • 每个master-slave都是一个哨兵系统, 即当master或者slave宕机的时候, 哨兵会知道是哪个宕机, 从而启动其他的slave机器;

  • 每个master-slave都有对应的槽位, 只存储对应槽位的数据;

步骤:

  1. 在根目录下新建一个文件夹: mkdir /myredis

  2. 在myredis文件夹下创建8个文件夹:

  3. 在每个文件夹里面放上一个redis.conf, 从redis自带的配置文件复制即可, 里面修改几个配置:

  1. 使用redis-server指定一个配置文件启动, 一个配置文件就是一个redis实例:

  2. 启动后使用ps -ef | grep reids查看是否都已经启动成功

  3. 开始搭建, 只需要一个命令:

    redis-cli -a mypasswd --cluster create ip:7000 ip:7001 ip:7002 ip:7003 ip:7004 ip:7005 ip:7006 ip:7007 --cluster-replicas 1

    说明: --cluster-replicas <arg> 这里的参数指的是 slave / master的值.

    这的1是slave与master的比值是1, 即master:slave = 1:1, 前面的四个是master, 后面的四个是slave.槽位区间自动平均分配;

    如果有9个redis实例, 这个值是2, slave:master=2:1, 前3个master, 后面六个是slave;

  4. 查看节点信息:

    注意后面的参数-c, 这里指的是cluster, 即以”集群”的方式进入redis

    至此, 搭建成功
