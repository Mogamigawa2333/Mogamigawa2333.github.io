逆波兰表示法

Posted on 2020-02-20

因为逆波兰表示法可以用栈来运算,所以更适合计算机;
像Lisp的那种前缀表示法也同理;
前缀和后缀没有括号来规定计算的优先级;
中缀表达式更适合人类阅读;

num_1 op num_2, num是数字或者参数, op是操作符或者函数.
转为后缀为: num_1 num_2 op

中缀 转 后缀:

基本思想:整体思想; a + b => a b +

示例: (手动)

a + b * ( c + d ) - e / f

  1. 首先把数字按顺序提取出来
    a b c d e f
  2. 找到优先级最高的一个整体,(c+d)或者e/f
    a b c d + e f
  3. 把 cd+ 看成一个数字, 再找剩下的优先级最高的一个整体,bcd+ 或者 e/f
    `a b c d +     e f`
  4. … …
    a b c d + * + e f
    a b c d + * + e f /
    a b c d + * + e f / -

===============================================================

基于以上的整体思想,可以修改一下:
a + b * ( c + d ) - e / f
先给中缀表达式根据运算顺序加上括号:
( ( a + ( b * (c + d ) ) ) - (e / f) )
然后从括号的最深处开始转化, c+d, b(c+d), a+`b(c+d), e/f,a+b*(c+d)+e/f`

  • 转化后的后缀表达式不一定是唯一的, 就如上面的所示,可以先算(c+d)也可以先算e/f,结果是不同的;

以上是手动的方式, 下面是栈的方式:
详见wiki: 调度场算法

简单描述调度场算法:
定义一个运算符栈

  1. 首先定义运算符的优先级
    括号() 大于 * / 大于 + -
  2. 左括号(一定有一个右括号)与之匹配; 所以遇到左括号时先入栈,遇到右括号时弹出所有从左括号到右括号内的运算符
  3. 优先级大的运算符在栈顶, 如果下一个运算符比栈顶的运算符的优先级小或者相等, 弹出栈顶的运算符
  4. 从左到右扫描算式.数字直接写出来.运算符进栈,根据以上规则弹出

第3条规则:
如果输入是一个数字则直接进输出队列;
如果输入是运算符，则压入操作符堆栈,
但是,如果输入运算符的优先级低于或等于运算符栈顶的操作符优先级,则栈内元素进入输出队列,输入操作符压入运算符堆栈.
最后，运算符堆栈内元素入输出队列，算法结束.

a + b * ( c + d ) - e / f
–> 从左至右扫描算式

逆波兰表示法的计算:

有一个数字栈,数字依次入栈,遇到操作符就弹出最上面两个数字与操作符进行计算,计算结果入栈.

计算 1 2 3 4 + * + 5 6 / - :

可以看出在转换的时候,用的栈是运算符栈.在计算的时候用的栈是数字栈;
